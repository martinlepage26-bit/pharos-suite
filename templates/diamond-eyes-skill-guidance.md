# Diamond Eyes Skill Guidance — Tier-Specific Application

Each tier applies Diamond Eyes to different outputs. This document shows how each tier validates before promoting work.

---

## Tier 0 (Forging) — Wisdom of Scope

**What you're validating:** Is the scope framing a problem in a way that serves genuine flourishing?

**Apply Diamond Eyes by asking:**

| Dimension | Question | Theater Risk |
| - | - | - |
| **Autonomy** | Does this scope preserve the ability of affected people to make choices about this? Or does it frame the problem in a way that assumes they have no choice? | Framing a problem as "unsolvable without X" removes autonomy from the solution. |
| **Transparency** | Are we naming what we DON'T know? What tradeoffs exist? Or are we claiming a neutral problem definition? | False neutrality ("this is just the way things are") hides that we made a choice. |
| **Accountability** | If this scope is wrong, who is responsible? Can they be held accountable? Or is scope-error diffused across governance layers? | Diffusing accountability ("everyone agreed to this scope") prevents anyone from being responsible for scope mistakes. |
| **Dignification** | Are the people affected by this scope treated as partners in defining it? Or as subjects whose needs we're solving for? | Treating vulnerable populations as subjects, not partners, removes their voice from the frame. |
| **Care** | Does this scope reflect genuine care for those affected? Or optimization of what's convenient to measure? | Optimizing for "measurable impact" while ignoring what people actually need is care theater. |

**Care Requirement:** Name explicitly: **What does this scope protect or enable for the vulnerable populations identified?**

If you cannot name what flourishing looks like for them, the scope is not wise.

**Decision Logic:**
- ✓ **PROMOTE** if: Scope is clear, transparent, preserves agency, acknowledges limitations, serves vulnerable populations' actual needs
- ⚠️ **CONDITIONAL** if: Scope is wise but assumes something that Governance should verify (e.g., "assumes data quality is X")
- ✗ **ESCALATE** if: Scope frames problem in a way that hides choice, removes agency, or optimizes metrics over people

## Tier 1 (Governance) — Wisdom of Constraints

**What you're validating:** Do governance constraints serve genuine flourishing or create the appearance of control while enabling capture?

**Apply Diamond Eyes by asking:**

| Dimension | Question | Theater Risk |
| - | - | - |
| **Autonomy** | Do constraints have exceptions? Who decides exceptions? Or is everyone bound uniformly, removing human judgment? | Constraints with no exceptions remove discretion. Constraints with secret exceptions remove accountability. |
| **Transparency** | Are constraints justified with reasoning people can understand and evaluate? Or are they rules that appear neutral? | Appearing neutral ("this is just the standard") hides the value choices embedded in constraints. |
| **Accountability** | If a constraint causes harm, can someone be held responsible? Or is the constraint itself treated as inevitable? | Treating constraints as inevitable ("we have to follow the rules") removes accountability from those who set them. |
| **Dignification** | Do constraints respect the dignity of those affected? Or do they treat people as inputs to be controlled? | Constraints that treat people as statistical problems, not humans, violate dignity. |
| **Care** | Do constraints reflect care for those governed? Or optimization of system performance? | Optimizing system efficiency while constraints harm vulnerable groups is care theater. |

**Care Requirement:** Name explicitly: **What does governance protect or preserve for those affected by these constraints?**

If constraints only benefit the governing system, not the governed, they are not wise.

**Decision Logic:**
- ✓ **APPROVE** if: Constraints are necessary, justified, transparent, reviewable, and protect those governed
- ⚠️ **CONDITIONAL APPROVAL** if: Constraints are wise but require conditions to prevent capture (e.g., "only if there is human review at X")
- ✗ **REJECT** if: Constraints are theater—they create appearance of control while enabling harm, or serve governance system more than governed population

## Tier 2 (Right-Arms) — Wisdom of Analysis

### Philosopher

**What you're validating:** Is this philosophical frame enabling genuine flourishing or rationalizing something already decided?

**Diamond Eyes check:**
- ✓ Does the frame illuminate genuine tension? Or does it defend one side?
- ✓ Does the frame make space for disagreement? Or does it claim to settle the matter?
- ✓ Can someone disagree with this frame based on different values? Or does it hide the value choice?

**Decision:** Does promoting this analysis serve wisdom, or does it end conversation prematurely?

### Power-Analyst

**What you're validating:** Does this power map expose real structure or distribute blame to obscure accountability?

**Diamond Eyes check:**
- ✓ Are vulnerable populations shown as affected? Or are they invisible in the power structure?
- ✓ Does the map show who benefits from the current structure? Or does it treat the structure as neutral?
- ✓ Does the analysis identify where change-points exist? Or does it imply the structure is immovable?

**Decision:** Does promoting this analysis serve those without power, or does it mystify power relations?

## Tier 3 (Research) — Wisdom of Evidence

**What you're validating:** Does this research methodology generate wisdom or just data that will be interpreted to justify power?

**Apply Diamond Eyes by asking:**

| Dimension | Question | Theater Risk |
| - | - | - |
| **Autonomy** | Are research subjects treated as partners in research design? Or as subjects whose behavior is studied? | Subject-treatment removes agency from those being studied. Partner-treatment means they influence what counts as important. |
| **Transparency** | Can the research be interpreted multiple ways? Are those interpretations acknowledged? Or does the research imply a single true answer? | Research implying a single answer serves power. Research acknowledging ambiguity enables wisdom. |
| **Accountability** | Who chose these research methods? Who benefits if results turn out a particular way? Can researchers be held accountable for methodology choices? | Methods chosen to produce particular results is corruption. Transparent methodology choice enables accountability. |
| **Dignification** | Are the populations being studied treated with respect? Are results presented in ways that affect them? | Publishing results about vulnerable populations without affecting their agency treats them as objects, not people. |
| **Care** | Does research design show care for research subjects? Or optimization of statistical power? | Optimizing statistical power while harming research subjects is care theater. |

**Care Requirement:** Name explicitly: **What does this research protect or enable for the populations it studies?**

If research serves the researcher's career more than it serves those studied, it's not wise.

**Decision Logic:**
- ✓ **PROCEED** if: Research design is transparent, subjects are partners, methodology is defensible, results will be shared with those affected
- ⚠️ **CONDITIONAL** if: Research is wise but requires IRB/ethics board approval of how subjects will be treated
- ✗ **REJECT** if: Research design serves researcher's convenience over subject welfare, or implies studying populations as if they have no voice

## Tier 4 (Writing) — Wisdom of Communication

**What you're validating:** Does this writing illuminate or obscure? Does it empower readers or manipulate them?

**Apply Diamond Eyes by asking:**

| Dimension | Question | Theater Risk |
| - | - | - |
| **Autonomy** | Can the reader understand this well enough to disagree? Or is it written to prevent disagreement? | Writing that prevents disagreement (by obscuring reasoning) removes reader autonomy. |
| **Transparency** | Are limitations named? Are alternatives acknowledged? Or does the writing claim certainty? | Claiming certainty where uncertainty exists is false authority. |
| **Accountability** | Can the author be held accountable for what they wrote? Or is it ambiguous enough to deny intent? | Ambiguity that protects the author from accountability is theater. |
| **Dignification** | Does the writing respect the reader's intelligence? Or does it treat them as needing to be led to a conclusion? | Treating readers as unable to think removes dignity. Presenting evidence and letting them decide preserves it. |
| **Care** | Does the writing show care for how readers will use it? Or optimization of author's message? | Optimizing for message control while ignoring reader impact is care theater. |

**Care Requirement:** Name explicitly: **What does clear writing protect or enable for readers?**

If writing serves the author's agenda more than reader understanding, it's not wise.

**Decision Logic:**
- ✓ **PROMOTE** if: Writing is clear, limitations acknowledged, reasoning transparent, readers can evaluate independently
- ⚠️ **CONDITIONAL** if: Writing is mostly clear but needs specific revisions to prevent misinterpretation
- ✗ **ESCALATE** if: Writing obscures limitations, claims false certainty, or is designed to manipulate rather than inform

## Tier 5 (Meta) — Wisdom of Architecture

**What you're validating:** Does this system design enable genuine flourishing or embed power asymmetries while claiming neutrality?

**Apply Diamond Eyes by asking:**

| Dimension | Question | Theater Risk |
| - | - | - |
| **Autonomy** | Can the system be gamed? Who can game it? Or is it robust for everyone? | Systems robust only for some users while others are vulnerable embed inequality. |
| **Transparency** | Can people understand how this system works? Or is it opaque? | Opacity prevents accountability. Clarity enables judgment. |
| **Accountability** | If the system harms someone, who is responsible? Or is harm treated as inevitable system behavior? | Treating harms as "just how the system works" removes accountability. |
| **Dignification** | Does the system treat all users with equal dignity? Or some as exceptions needing special handling? | Systems that require "special handling" for some users treat them as less-than. |
| **Care** | Does the system show care for people affected? Or optimization of system metrics? | Systems optimized for metrics while ignoring human impact are care theater. |

**Care Requirement:** Name explicitly: **What does this system design protect or enable for all users?**

If the system serves some users' interests more than others', it's not wise.

**Decision Logic:**
- ✓ **PROMOTE** if: System is transparent, robust for all, preserves user agency, handles exceptions with dignity
- ⚠️ **CONDITIONAL** if: System is wise but requires safeguards to prevent certain failure modes
- ✗ **ESCALATE** if: System design enables gaming, removes user agency, or treats inequality as inevitable

## Quick Reference: Diamond Eyes Red Flags

**ALWAYS escalate if you notice:**

- [ ] Claiming neutrality on choices that are actually values-based
- [ ] Distributing accountability so no one is responsible
- [ ] Treating vulnerable populations as subjects, not partners
- [ ] Optimizing metrics while ignoring human impact
- [ ] Making exceptions secret or special-cased
- [ ] Framing hard choices as inevitable
- [ ] Removing human judgment in favor of rules
- [ ] Theater: appearance of control while enabling harm

**ALWAYS promote if you can verify:**

- [ ] Value choices are named and debated
- [ ] Accountability is clear and real
- [ ] Vulnerable populations have voice in decisions
- [ ] Limitations are acknowledged
- [ ] Exceptions are transparent
- [ ] Human judgment is preserved
- [ ] Care is visible, not theater
