# Website Protocol

Use this when building website branding directions or critiquing a site.

## Homepage hierarchy

The homepage should answer, in order:
1. what this is
2. who it is for
3. why it is credible
4. what the user should do next

## Navigation tone

- Keep labels plain and legible.
- Avoid cleverness that obscures where information lives.
- In trust-heavy categories, clarity beats novelty.

## Layout rhythm

- Use whitespace to control seriousness and pace.
- Keep the hero dense enough to establish authority without feeling cramped.
- Avoid equally weighted sections that flatten importance.

## Trust architecture

Show evidence in the right places:
- credentials
- method
- cases or scenarios
- deliverables
- process

Trust should feel inspectable, not merely asserted.

## Typography system

- Use a display/utility split only when each role is clear.
- Ensure body text stays calm and readable.
- Avoid type combinations that look editorially ambitious but operationally fragile.

## Image logic

- Images should either prove, orient, or clarify.
- Decorative technology imagery is a liability in serious categories.
- Diagrams are often more credible than stock photography.

## Interaction tone

- Motion should guide attention, not create spectacle.
- Calls to action should feel decisive, not salesy.
- Hover states and glows should support hierarchy, not become the main event.

## Mobile legibility

- Re-check hierarchy on mobile.
- Keep spacing disciplined.
- Preserve the credibility signal under compression.
