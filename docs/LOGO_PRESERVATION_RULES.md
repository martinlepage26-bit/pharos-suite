# LOGO PRESERVATION RULES

Effective date: 2026-03-25

## Fixed Elements (Must Remain Literal)

1. First green band as canonical identity/header band.
2. Diamond logo mark centered within the canonical composition.
3. Shiny soft orb/light above the diamond, preserved as part of the full logo composition.
4. Relative hierarchy: band as field, diamond as primary mark, orb as integrated crown/light element.

## Allowed Adaptations (Without Restyling)

- Uniform scaling of the full composition.
- Proportional cropping only when the full mark cannot physically fit and the adaptation is labeled constrained-space.
- Export-format changes (PNG/SVG/WebP) without visual redesign.
- Padding and placement adjustments for responsive layouts.

## Prohibited Alterations

- Recoloring the canonical green band.
- Replacing the diamond mark with a generic geometric substitute.
- Flattening or removing the orb/light in full-logo contexts.
- Hardening the orb glow into a harsh/neon or unrelated effect.
- Replacing the band with a neutral strip or different band treatment.
- Presenting alternate marks as if they were canonical.

## Usage Guidance

### Full Logo Use

Use the full canonical composition (green band + diamond + orb) for primary identity placements, hero headers, and first-impression brand moments.

### Header Band Use

When a header strip is used, the first green band treatment is authoritative. Keep the diamond and orb relationship intact in standard header contexts.

### Constrained-Space Use

If space is too small for full composition legibility:

- Prefer a carefully scaled canonical lockup first.
- If reduction is required, produce an explicitly labeled derivative variant.
- Do not silently drop the orb in canonical contexts.

### Favicon/App-Icon Adaptation

A favicon/app icon may use a constrained derivative due to extreme size limits. It must be labeled derivative and mapped back to the canonical lockup in documentation.

### Dark/Light Background Handling

Preserve band, mark, and orb relationship. Adjust surrounding layout/background treatment, not the canonical logo internals.
