# Diagnostic Protocol

Use this when critiquing an existing identity, logo, or website.

## First read

- State what the identity signals in the first 3 seconds.
- State the likely category read.
- State whether the signal helps or harms the offer.

## Strategic fit

- Match the identity against the offer, audience, and buying context.
- Identify where the identity overpromises, under-signals, or misclassifies the business.

## Audience read

- Describe what a likely buyer infers.
- Separate target-audience inference from non-target-audience inference.
- Separate founder intent from buyer interpretation.

If the target and non-target audience read the identity the same way, the identity may be too generic or poorly positioned.

## Formal issues

Check:
- hierarchy
- contrast
- geometry
- complexity
- distinctiveness
- optical balance
- typographic control

## System issues

Check:
- monochrome behavior
- small-size legibility
- favicon or avatar behavior
- slide and PDF behavior
- website extension
- repeatability by other designers

## Revision path

End with:
- preserve
- remove
- simplify
- elevate
- test next

Order the next moves by impact. Fix the signal and positioning errors before polishing minor formal details.
