# Diagnose Rule Failure

Use this file when people keep breaking, ignoring, or working around a rule.

## Starting Assumption

Start with Rule Five:

- If many people break a rule, suspect the rule design before assuming widespread bad intent.

Design for willing people. Govern the willful minority separately.

## Fast Diagnostic Sequence

1. Describe the exact behavior that is missing or unwanted.
2. Check whether people can comply.
3. Check whether people have the opportunity to comply.
4. Check whether they are motivated to comply.
5. Check whether attention, overload, or habit is blocking compliance.
6. Check whether incentives, norms, or management signals undermine the rule.
7. Escalate to misconduct only after testing the earlier explanations.

## Framework Shortcuts

### RADAR

Use to decide whether the issue is isolated or systemic and to spot where collective behavior is heading.

Good for:

- repeated workarounds
- early-warning trend detection
- deciding whether to redesign the rule

Sequence:

1. Recognize the exact behavior gap.
2. Analyze the likely causes using COM-B, TDF, or adjacent lenses.
3. Design the lightest intervention that addresses the real cause.
4. Act by naming an owner, timeline, and support needed.
5. Review with a concrete signal such as observation, audit, or behavior change.

### COM-B

Use to test:

- Capability: Do people know how to comply, and can they do it in practice?
- Opportunity: Do they have the tools, time, workflow, environment, and social support?
- Motivation: Do incentives, values, habits, and local norms support the rule?

Interpretation:

- Capability failures call for training, simplification, or clearer examples before punishment.
- Opportunity failures call for workflow, tooling, staffing, or environment redesign.
- Motivation failures call for legitimacy work, incentive realignment, feedback, or consequences.

### TDF

Use when COM-B is too coarse and the user needs deeper causal categories such as social influence, environmental context, role identity, or beliefs about consequences.

### MSI

Use when the issue looks like real-world bounded behavior rather than explicit resistance.

- Motivation problem
- Self-control problem
- Inattention problem

### System model

Use when the rule sits inside trust, culture, community, and psychological ownership rather than only formal compliance mechanics.

## Response Logic

- If capability is missing, redesign training, examples, or documentation.
- If opportunity is missing, redesign tooling, workflow, time, access, or environment.
- If motivation is missing, redesign incentives, meaning, peer norms, or feedback loops.
- If inattention is the issue, simplify, sequence, prompt, or automate.
- If behavior is clearly willful, tighten governance, monitoring, and consequences.

## Ethics Lens

For ethics-heavy domains, anchor the response in visible values instead of generic prohibition language.

Example healthcare values:

- care
- compassion
- courage
- communication
- competence
- commitment

Also apply Rule Six:

- Just because you can, does not mean you should.

## Useful Output Format

- Observed failure
- Likely cause
- Evidence or signal
- Human-centered redesign
- Governance backstop for willful misuse
